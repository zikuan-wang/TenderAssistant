## 1. 项目现状

TenderAssistant 当前实现是一个本地单机版 WPF 投标资料辅助工具。交接信息以当前代码、配置和打包脚本为准；历史 Word 方案文档已删除，不再作为当前项目边界依据。现有代码没有后端服务、API 服务、数据库、对象存储或网络同步流程，源码边界已经收敛为本地客户端、离线授权工具和授权模型库。

当前可用能力：

- 本地标书资料库浏览、筛选、预览和插入。
- 支持 PDF、图片、Word/RTF、TXT 资料插入到已打开的 Word/WPS 文档光标位置。
- PDF 可按质量档位渲染为图片，可选择全部页面或前 N 页 + 后 N 页插入。
- 本地文件库从源目录同步到客户端缓存目录。
- 本地文件库文件可从界面打开、定位到资源管理器位置，也可删除本地缓存副本。
- 用户自定义导入文件列表持久化。
- 离线授权申请导出、离线激活文件导入和本机授权校验。
- 授权生成器可导入申请 JSON 并生成 `.bali` 激活文件。
- 客户端操作日志、归档、导出、清空。
- 主题切换：浅色、深色、粉色。
- GitHub Release 在线检查更新和 MSI 下载启动。
- MSI 安装包打包到 `Program Files\TenderAssistant`。

当前未实现能力：

- 服务端后台、登录、组织/部门管理、数据库。
- 远程文件库、在线授权、在线价格库、审批流。
- 自动生成完整投标文件。
- 自动化测试项目。

### 5.1 标书页

核心文件：

- `src/TenderAssistant.Client/Views/Pages/BidAssistPage.cs`
- `src/TenderAssistant.Client/Views/Pages/BidAssistPageView.xaml`
- `src/TenderAssistant.Client/ViewModels/Pages/BidAssistPageViewModel.cs`

- `src/TenderAssistant.Client/Services/OfficeDocumentInsertService.cs`
- `src/TenderAssistant.Client/Services/PdfPageRenderService.cs`

文件分类：

- `technical`：技术文件。
- `business`：商务文件。
- `qualification`：资质文件。
- `custom`：自定义导入，不复制到本地文件库，只保存文件路径。

源文件库同步规则：

- 源目录下识别这些候选目录名：
  - 技术：`technical`、`技术文件`、`技术`
  - 商务：`business`、`商务文件`、`商务`
  - 资质：`qualification`、`资质文件`、`资质`
- 同步为递归复制。
- 跳过 `.ta-meta.json` 文件。
- 同名目标文件存在且长度相同、目标修改时间不早于源文件时跳过。
- 注意：当前 `LoadLibraryFiles()` 只枚举 `technical`、`business`、`qualification` 分类目录顶层文件；同步过程会递归复制子目录，但子目录中的文件不会出现在标书页列表中。

支持文件类型：

- PDF：`.pdf`
- 图片：`.png`、`.jpg`、`.jpeg`、`.bmp`、`.gif`、`.tif`、`.tiff`
- Word/RTF：`.doc`、`.docx`、`.rtf`
- 文本：`.txt`

插入逻辑：

- 插入前必须通过本机授权校验。
- 目标文档必须已经在 Word 或 WPS 中打开，并且光标定位到目标位置。
- COM ProgID 尝试顺序：
  - `Word.Application`
  - `KWPS.Application`
  - `WPS.Application`
  - `kwps.Application`
  - `wps.Application`
- 图片按用户设置宽度插入，默认 `14cm`。
- TXT 按 UTF-8 纯文本插入。
- DOCX 可选择保留格式或抽取正文纯文本插入。
- PDF 优先通过 Docnet 渲染为 PNG 后逐页插入，失败时尝试作为 OLE 对象嵌入。
- PDF 页面图片缓存按文件完整路径、长度、最后修改时间、页码和 DPI 生成 SHA-256 文件名；当前没有自动清理策略。
- PDF 质量档位：
  - 标准模式：150 DPI。
  - 压缩模式：96 DPI。
  - 高清模式：200 DPI。

## 6. 授权机制

共享授权模型在：

- `src/TenderAssistant.Licensing/OfflineLicenseModels.cs`

客户端授权服务在：

- `src/TenderAssistant.Client/Services/OfflineLicenseService.cs`
- `src/TenderAssistant.Client/Services/DeviceFingerprintService.cs`

授权生成器在：

- `src/TenderAssistant.Authorizer/MainWindow.xaml`
- `src/TenderAssistant.Authorizer/MainWindow.xaml.cs`
- `src/TenderAssistant.Authorizer/OfflineLicenseIssuer.cs`

离线授权流程：

1. 客户端设置页导出授权申请 JSON。
2. 授权申请包含申请编号、申请人、机器名、设备指纹、申请时间。
3. 授权生成器导入申请 JSON。
4. 选择授权有效期和版本，例如 `Standard`。
5. 授权生成器生成 `.bali` 激活文件。
6. 客户端导入 `.bali`，复制到客户端程序目录。
7. 客户端每次关键操作前校验授权文件。

设备指纹组成：

- `Environment.MachineName`
- `Environment.UserDomainName`
- WMI `Win32_ComputerSystemProduct.UUID`
- WMI `Win32_BIOS.SerialNumber`
- 上述内容拼接后做 SHA-256，取前 32 位十六进制字符串。

授权文件格式：

- 授权载荷由授权生成器私钥做 RSA SHA-256 签名。
- `.bali` 文件以 `BALI1.` 开头。
- 授权 envelope 使用 AES-GCM 加密。
- 客户端持有公钥并验签。
- 当前 `KeyId`：`offline-rsa-2026-05`。

重要安全事项：

- `src/TenderAssistant.Authorizer/OfflineLicenseIssuer.cs` 内置了授权私钥。
- 授权生成器和源码仓库需要按敏感资产管理，不应随客户端安装包公开分发。
- 客户端 `TenderAssistant.Licensing` 中包含公钥和激活文件 AES 派生常量，这是验签和读取激活文件所必需的客户端侧逻辑。

## 9. 当前代码质量与风险

必须关注：

- 授权私钥硬编码在授权生成器源码中，仓库和 Authorizer 产物都应限制访问。
- 没有自动化测试，尤其是授权验签、PDF 页选择、文件库同步、Word/WPS COM 插入都缺少回归测试。
- 客户端插入依赖本机 Office/WPS COM 可用，且目标文档必须已打开。
- PDF 文本预览是轻量提取，不等价于完整 PDF 解析。
- 源文件库同步会递归复制子目录，但当前标书页只加载分类目录顶层文件，嵌套目录资料可能同步成功却不可见。
- PDF 渲染缓存没有自动清理，长时间大量插入 PDF 后 `%LOCALAPPDATA%\TenderAssistant\PdfPageCache` 可能持续增长。
- `artifacts/` 是本地生成目录，不应作为源码状态依据。
- 版本号目前需要手动同步 `Directory.Build.props` 和 WiX `Product.wxs`。
- 当前实现是本地版；如后续重启服务端规划，需要基于当前代码重新整理需求边界。

建议后续补齐：

- 建立最小测试项目，优先覆盖 `TenderAssistant.Licensing`、`BidAssistFileCatalogService`、`PdfPageRenderService.GetSelectedPageIndexes` 的可测试逻辑。
- 把授权私钥移出源码，至少改为受控本地机密或专用签发环境。
- 增加 Authorizer 发布脚本，和客户端 MSI 发布流程分开管理。
- 增加版本号同步脚本或 MSBuild 属性注入，降低发布时漏改风险。
- 在 README 中补充 MSI 发布、授权生成器分发和 GitHub Release 命名约定。
- 如果重新启动服务端规划，需要先明确当前本地版和未来联网版的数据边界，避免把价格、授权和文件库策略混在客户端。

## 10. 接手检查清单

新接手者建议按以下顺序确认：

1. `dotnet --info` 确认 SDK 版本满足 `global.json`。
2. `dotnet restore TenderAssistant.sln`。
3. `dotnet build TenderAssistant.sln`。
4. 启动客户端，确认标书页和设置页能打开。
5. 启动 Word 或 WPS，打开一个测试文档并定位光标。
6. 用授权生成器完成一次申请导出、激活生成、客户端导入。
7. 配置一个源文件库，目录中放 `技术文件`、`商务文件`、`资质文件`。
8. 在设置页同步文件库。
9. 插入 TXT、图片、DOCX、PDF 各一个样例。
10. 执行 `.\packaging\TenderAssistant.Client.Wix\build-msi.ps1 -Configuration Release`。
11. 安装 MSI，确认开始菜单、桌面快捷方式、安装目录和授权文件导入路径。
